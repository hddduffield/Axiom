## FORMATTING RULES

### Numbers
- Use bold for emphasized amounts in text and tables
- Use $XXX,XXX format for amounts under $1M; $X.XM for amounts $1M-$999M; $XB for billion
- Don't mix formats in the same section
- Round consistently: 2 decimals for rates, 0 decimals for dollars in summary contexts

### Dates
- "April 15, 2027" not "4/15/27" or "April 15"
- "Q1 2027" or "Q1 of 2027"
- "Year T-2" (for pre-transaction sequencing where transaction date is T)

### Citations
- Use end-references (Section 16. References) rather than parenthetical (IRC §1202(d))
- Inline citations for foundational concepts: "under §1202..."
- Footnotes only for genuinely needed contextual notes

### Lists
- Use numbered lists for ordered actions
- Use bullet lists for parallel items without ordering
- Limit nested lists; if you need 3 levels of nesting, restructure

### Bold and italic
- Bold for terms-of-art on first use, then plain
- Bold for emphasized numerical values
- Italic for actual phrases in trust language, statute names, case names

### Headers
- Title (level 1): plan title
- Section (level 2): "1. Executive Summary", "2. Your Situation", etc.
- Sub-section (level 3): individual recommendations or topics within a section
- Sub-sub (level 4): rare; mostly for procedural steps

### Footers
- Page number / total pages
- "Prepared by PSA Wealth" or compliance line
- Compliance ID on each page

## BAD FORMATTING — RED FLAGS

- 5+ heading levels (collapse hierarchy)
- Page-long bullet lists (convert to prose or table)
- Tables that span 3+ pages (split or convert)
- Inline parentheticals everywhere (§1202(d)(2)(B)) (move to references)
- Mixed citation styles (pick one)
- Numbers as words and digits inconsistent (pick one for each context)
