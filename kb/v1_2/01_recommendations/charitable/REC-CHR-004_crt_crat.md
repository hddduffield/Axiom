# [REC-CHR-004] — Charitable Remainder Annuity Trust (CRAT)

## TRIGGERING CONDITIONS

### Structured logic
```
TRIGGER if ALL of:
  - Same general triggers as CRUT
  - Donor wants FIXED annual amount (not %-based)
  - Higher §7520 favorable for CRAT (vs. CRUT favors lower)

DISQUALIFY if:
  - 5% probability test fails (below current §7520 rates rare; possible if rates spike)
  - 10% remainder test fails
```

### Natural-language explanation
A CRAT pays a FIXED annual annuity (not percentage of changing value) to non-charitable beneficiary, with remainder to charity. Predictable income stream; less common than CRUT for most planning. Sensitive to §7520 rate.

### Hard disqualifiers
- 5% probability test failure under current rate environment
- 10% remainder test failure

## WHAT IT IS
Same general structure as CRUT but with fixed annuity payment instead of variable unitrust amount. No annual revaluation needed; payment amount determined at funding.

## WHY WE RECOMMEND IT (when triggered)
For donors valuing predictability of income over inflation protection. Mathematically advantaged when §7520 is high.

## QUANTIFIED IMPACT FRAMEWORK
Same components as CRUT but fixed annuity calculation.

## IMPLEMENTATION STEPS
Same as CRUT.

## SEQUENCING DEPENDENCIES

## DOCUMENTATION CHECKLIST

## COMMON MISTAKES
- 5% probability test failure (must show <5% chance trust runs out before charity)
- 10% remainder test failure
- Failing fixed-amount discipline if asset returns volatile

## COORDINATION NOTES
Same as CRUT.

## CLIENT CONVERSATION FRAMING
> "CRAT pays you a fixed annual amount instead of a percentage of trust value. Predictable, but no inflation hedge. We typically recommend CRUT instead unless predictability is the priority."

## CAVEATS & DISQUALIFIERS
- 5% probability test
- 10% remainder test
- Less inflation protection than CRUT
